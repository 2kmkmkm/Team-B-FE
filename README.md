# 🎁 Gifty  
모바일 기반 선물 펀딩 서비스

> 구름톤 유니브 전남대 2기 미니 프로젝트  
> “받기 편한 선물”이 아닌, **정말 받고 싶은 선물**을 위한 서비스


## 📌 Overview
GIFty는 **선물을 받는 사람이 직접 펀딩을 개설**하는 모바일 기반 선물 펀딩 서비스입니다.  
기존 선물하기 서비스에서 발생하는 *원치 않는 선물* 문제를 해결하고자,  
받는 사람이 원하는 선물을 명확히 제시하고 지인이 함께 펀딩에 참여하는 구조로 기획되었습니다.

본 프로젝트는 **프론트엔드 개발자로 참여한 첫 팀 프로젝트**로,  
React 기반 컴포넌트 설계와 협업 개발 프로세스를 실전처럼 경험하는 것을 목표로 진행했습니다.

- axios를 활용한 **비동기 API 통신 구현**
- 기획안 기반 화면 흐름 정리 및 UX 구현


## 🛠 Tech Stack
### Frontend
- **React**: 컴포넌트 기반 UI 구성
- **JavaScript**
- **Styled-Components**: 컴포넌트 단위 스타일 관리

### Build & Deploy
- **Vite**: 빠른 개발 환경 구성
- **Vercel**: 배포 및 테스트 환경 구성

### Collaboration
- **GitHub**: 브랜치 기반 협업
- **Notion**: 회의록 및 기획 공유
- **Discord**: 팀 커뮤니케이션


## ✨ Key Features
- **회원가입 및 로그인**
  - 이메일/비밀번호 기반 인증
  - 로그인 후 개인화된 펀딩 정보 확인

- **선물샵 연동 펀딩 개설**
  - 원하는 선물을 선택해 펀딩 생성
  - 목표 금액, 마감일 설정

- **펀딩 참여**
  - 친구 목록을 통해 펀딩 확인
  - 참여 금액 직접 입력 후 기여

- **펀딩 리스트 및 상세 페이지**
  - 전체 펀딩 현황 카드 UI 제공
  - 펀딩별 상세 정보 확인


## 🔍 What I Focused On
- **컴포넌트 단위 UI 설계**
  - 레이아웃과 UI를 재사용 가능한 컴포넌트로 분리하여 구현
- **UX 흐름 이해**
  - 단순 기능 구현이 아닌,  
    *펀딩 개설자 / 참여자* 관점에서 화면 전환과 사용 흐름을 고려
- **협업 경험**
  - GitHub 브랜치 전략과 Notion 문서 공유를 통해  
    개발 외 영역에서도 팀 협업의 중요성을 체감


## 🌱 What I Learned
- React와 Styled-Components를 **실무 흐름에 가깝게 처음 사용**하며  
  컴포넌트 구조와 props 활용에 대한 이해도가 크게 향상됨
- 기술 구현뿐 아니라 **기획과 UX가 명확하지 않으면 사용자에게 혼란을 줄 수 있음**을 실전에서 경험
- 기술 역량이 부족하더라도 **협업을 통해 유의미한 결과물을 만들 수 있음**을 배운 첫 프로젝트
